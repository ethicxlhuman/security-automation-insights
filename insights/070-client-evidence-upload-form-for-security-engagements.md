# Client Evidence Upload Form for Security Engagements

Photo-based project intake form that enforces minimum artifact uploads and full field validation before submission, built for cybersecurity service firms.

## The Problem

Security firms waste hours on intake chaos:
- Clients email screenshots as attachments with no context
- Pentest scope docs arrive incomplete, missing environment details
- GRC evidence packages submitted with wrong files or missing assets
- No validation gate means bad submissions reach the analyst queue

**Result:** Analysts spend 2-4 hours per engagement just cleaning up intake.

## The Solution

A structured intake form that enforces:
- Minimum 3 file uploads with client-side preview before submission
- Required fields: name, email, service type, project description
- Submit button locked until all conditions are met
- Preview grid so client confirms correct files before sending

No bad submissions reach the queue.

## Use Cases

- Pentest firms collecting environment screenshots and scope artifacts
- GRC consultants receiving client evidence packages for SOC 2 / ISO 27001
- MDR onboarding: clients upload existing SIEM configs and alert samples
- Cloud security assessments: clients submit architecture diagrams

## Impact

- Eliminates incomplete intake submissions reaching the analyst queue
- Reduces intake processing time by 60-80% per engagement
- Client confirms correct files via preview before sending
- Deployable as a standalone page or embedded in existing client portal
