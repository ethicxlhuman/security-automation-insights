# Cybersecurity Site Search Engine

Prototype: https://ai-website-cybersecurity-company.vercel.app/

## The Problem

Security company websites bury critical content — service pages, compliance docs, case studies — with no way for prospects to find it fast. Standard nav menus fail when you have 20+ pages.

## The Solution

- Client-side Fuse.js index built at page load (zero backend)
- Instant results on every keystroke, no search button
- Keyword highlighting in titles, descriptions, and body content
- Filters scoped to security subsegments (GRC, MDR, Cloud, IAM)

## Use Cases

- Prospects searching for specific compliance coverage (SOC 2, HIPAA)
- Security buyers comparing service offerings mid-visit
- Internal teams navigating large documentation sites

## Tech Stack

- Fuse.js (fuzzy search, client-side)
- Vanilla JS / React
- No database, no API calls, no auth required

## Target Audience

Security founders and CTOs running content-heavy cybersecurity sites

## Impact

Reduces prospect drop-off from navigation friction. Keeps buyers on-site longer — especially useful when pitching to CISOs with 3 minutes to evaluate you.
