# Service Package Builder

Prototype > https://ai-website-cybersecurity-company.vercel.app/package-builder

A dynamic, rule-enforced service configuration tool that lets prospects self-select and validate cybersecurity service packages in real time, then share or restore their configuration via URL state.

## Problem

Cybersecurity firms waste significant time on scoping calls where prospects don't know what they need. Sales teams manually walk clients through service tiers, combinations, and pricing — every time. There's no self-serve layer, no guardrails, and no persistent state between conversations.

## Solution

A browser-based package builder with:
- Category-based service selection across MDR, GRC, cloud, and support tiers
- Live package summary with running cost estimate
- Combination rules enforced before the package is finalized (required pairs, blocked conflicts)
- Shareable URL state so prospects can resume or send a pre-configured package to sales

## Use Cases

- **MDR/SOC firms**: Let prospects configure their alert triage + reporting + support tier before the first call
- **GRC consultancies**: Guide clients through compliance scope selection (SOC 2 + ISO 27001 + HIPAA) with dependency rules enforced
- **Offensive security teams**: Package recon, web/API testing, and report delivery as selectable modules
- **Sales enablement**: Replace PDF pricing sheets with interactive, shareable configurations

## Impact

- Reduces average scoping call time by eliminating back-and-forth on basic package selection
- Increases deal qualification quality — sales receives a pre-validated config, not a blank slate
- URL-based state persistence means no lead data loss between sessions
- Rule enforcement prevents invalid or unprofitable combinations from reaching the pipeline
